Modal Persistence Fix Applied

✅ Root Cause Identified & Fixed:

🐛 The Problem:

The closeAllModals() function was removing the main event modal from the DOM entirely, instead of just hiding it. After the first close, the modal element no longer existed.

🔧 The Solution:

Modified closeAllModals() to preserve the main event modal by checking:

if (modal.id !== 'eventModal' && !modal.querySelector('#eventModal'))

🧪 Test Scenario:

1. Click "New Event" or click on calendar date

2. Modal opens with office dropdown populated ✅

3. Close modal (X button, Cancel, or Escape key)

4. Click "New Event" again

5. Modal should open again successfully ✅

6. Repeat steps 3-5 multiple times

Expected Result: Modal works consistently on every open/close cycle. No more "Event modal not found!" errors.
