🔧 JavaScript Fixes Applied

Problem Identified:

The calendar JavaScript was using toLocaleTimeString('en-US') without specifying timezone, causing it to display times in the user's browser timezone instead of Philippines timezone.

Solution Applied:

Added timeZone: 'Asia/Manila' to all JavaScript date formatting functions:

  • ✅ Event hover tooltips
  • ✅ Event details modals
  • ✅ Conflict detection displays
  • ✅ Calendar header updates
  • ✅ Export functions

Code Changes:

// Before (Browser timezone):
startDate.toLocaleTimeString('en-US', { 
    hour: 'numeric', 
    minute: '2-digit',
    hour12: true 
})

// After (Philippines timezone):
startDate.toLocaleTimeString('en-US', { 
    hour: 'numeric', 
    minute: '2-digit',
    hour12: true,
    timeZone: 'Asia/Manila'  // ← Added this
})
