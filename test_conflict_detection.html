Calendar Conflict Detection System

✅ Conflict Detection Features Implemented

Backend Implementation:

  • ConflictResolverService - Core conflict detection logic
  • ConflictResolution - Structured conflict response
  • EventRepository - Database conflict queries
  • API Endpoint - /events/api/check-conflicts
  • Role-based permissions - OSEC/Admin can override
  • Detailed conflict info - Event details, times, locations

Frontend Implementation:

  • Real-time checking - As user types start/end times
  • Visual indicators - Warning badges in form
  • Conflict details modal - Shows conflicting events
  • Override capability - For authorized users
  • Debounced requests - Efficient API calls
  • Form validation - Prevents submission if blocked

🔄 Conflict Detection Flow

1

User Input

User selects start and end times in the event modal

2

Real-time Check

System automatically checks for conflicts (debounced 500ms)

3

Conflict Detection

Backend queries database for overlapping events

4

Visual Feedback

Warning indicator appears with conflict count and details

5

Resolution

User can view details, cancel, or override (if authorized)

👥 User Roles and Conflict Resolution

Regular Users

  • • Cannot override conflicts
  • • Must resolve conflicts first
  • • Can view conflict details
  • • Form submission blocked

OSEC Users

  • • Can override conflicts
  • • See warning messages
  • • "Proceed Anyway" option
  • • Full conflict details

Admin Users

  • • Full override permissions
  • • Can force event creation
  • • Access to all conflict data
  • • System management rights

⚙️ Technical Implementation Details

Backend API Endpoint:

POST /events/api/check-conflicts 
{
  "start": "2026-02-14T10:00",
  "end": "2026-02-14T11:00",
  "exclude_event_id": null
}

API Response Format:

{
  "success": true,
  "has_conflicts": true,
  "conflict_count": 2,
  "can_override": true,
  "conflicts": [
    {
      "id": 123,
      "title": "Team Meeting",
      "start": "2026-02-14T10:30:00+00:00",
      "end": "2026-02-14T11:30:00+00:00",
      "location": "Conference Room A",
      "creator": "admin@tesda.gov.ph",
      "office": {
        "id": 1,
        "name": "Main Office",
        "code": "MAIN"
      },
      "priority": "high",
      "status": "confirmed"
    }
  ],
  "warning_message": "This time slot conflicts with 2 existing events. Do you want to proceed?"
}

Frontend Features:

  • • Debounced API calls - 500ms delay to prevent excessive requests
  • • Real-time indicators - Visual feedback as user types
  • • Detailed conflict modal - Shows all conflicting event information
  • • Role-based UI - Override button only for authorized users
  • • Form validation - Prevents submission for blocked conflicts
  • • Toast notifications - Success/error feedback
