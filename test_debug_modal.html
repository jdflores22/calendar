🔍 Enhanced Debugging Added:

Console Logging: Added detailed console.log statements to track modal opening
Element Detection: Multiple fallback methods to find office select

🧪 Debug Steps:

1. Open calendar page and Developer Console (F12)

2. Click "New Event" or click on a calendar date

3. Watch console for detailed debug messages:

  • "Opening event modal..."
  • "Modal shown"
  • "Setting up modal content..."
  • "Starting loadOfficesForModal..."
  • "Event modal found: true/false"
  • "Event form found: true/false"
  • "Office select found by ID: true/false"

4. If office select is not found, check the HTML preview in console

🚨 Possible Issues:

Template Compilation: The Twig template might not be rendering the select element

CSS/JS Conflicts: Other scripts might be interfering with the modal

Form Reset: The form might be getting reset before we can access elements

Duplicate IDs: There might be duplicate office_id elements
